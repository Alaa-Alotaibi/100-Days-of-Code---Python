# Rules for Learning to Code

## 1. Start small
Do not wait for the perfect moment. Even 20 minutes a day matters.

## 2. Build projects
Learning is easier when you make something real.

## 3. Understand the code
Do not just copy. Know what each line does.

## 4. Ask for help wisely
Use Google, documentation, and Stack Overflow when needed.

## 5. Copy to learn
Start by copying small projects, then change them.

## 6. Stay accountable
Track your progress and show your work.

## 7. Keep learning
Programming changes, so keep improving.

## 8. Take breaks
Sometimes the best solution comes after stepping away.

## 9. Find a mentor
Learning with help can speed up your progress.

## 10. Break problems into chunks
Big projects become easier when split into small steps.

## 11. Never copy code blindly
Test, remove, and understand every line.

## 12. Use GitHub to learn
Study other projects, compare solutions, and reverse engineer small apps.
